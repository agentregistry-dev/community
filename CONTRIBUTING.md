## Ways to contribute

Thanks for your interest in contributing to agentregistry! We have a few different ways you can get involved.

### Report Security Vulnerabilities

Information about common vulnerabilities and exposures (CVEs) can be found on the [agentregistry documentation site](https://aregistry.ai/docs/reference/vulnerabilities/)

### File issues

To file a bug or feature request in the [agentregistry GitHub repo](https://github.com/agentregistry-dev/agentregistry):

1. Search existing issues first.
2. If no existing issue addresses your case, create a new one.
3. Use issue templates when available.
4. Add information or react to existing issues, such as a thumbs-up 👍 to indicate agreement.

### Find something to work on

The project uses [GitHub issues](https://github.com/agentregistry-dev/agentregistry/issues) to track bugs and features. Issues labeled with the `good first issue` label are a great place to start.

Additionally, the project has a [kanban board](https://github.com/orgs/agentregistry-dev/projects/3) for the upcoming releases. Any issues in the `ready` column are a great source of things to work on.

## Contributing code

Contributing features to agentregistry is a great way to get involved with the project. We welcome contributions of all sizes, from small bug fixes to large new features.

Please see the project [contributing guide](https://github.com/agentregistry-dev/agentregistry/blob/main/CONTRIBUTING.md) for instructions on how to best contribute code.
